# Solarseed TOU Energy Metering

Time-of-use energy cost tracking for Home Assistant.

Home Assistant tracks your energy **consumption** — this integration tracks your energy **costs** by applying the right rate at the right time.

## What It Does

- Multiplies your energy sensor's kWh by the correct effective rate for the current time of day
- Computes effective rates using the full formula: `(tier_rate + regulatory + passthrough + programs) × (1 + tax / 100)`
- Handles multiple rate tiers (off-peak, on-peak, mid-peak, etc.)
- Supports seasonal rate schedules (summer vs. winter pricing)
- Recognizes holidays with configurable holiday rates
- Tracks fixed monthly charges separately
- Creates sensors: current rate, current tier, fixed monthly, cost/hr, cost today, cost this week, cost this month

## How Rate Configuration Works

Rate schedules are generated by the **[Johnny Solarseed Rate Calculator](https://johnnysolarseed.org/tou-calculator)** on the website. The calculator:

1. Walks you through your bill's component charges (usage, transmission, distribution, regulatory, taxes, etc.)
2. Computes summed formula values for each rate tier
3. Generates a complete YAML configuration

You paste the YAML into the integration's **Options** flow. No in-plugin GUI is needed — the website handles all rate decomposition, schedule painting, and bill validation.

## Installation

### HACS (Recommended)
1. Open HACS → Integrations → Custom Repositories
2. Add: `https://github.com/danrichardson/solarseed-tou-metering`
3. Install "Solarseed TOU Energy Metering"
4. Restart Home Assistant

### Manual
Copy `custom_components/solarseed_tou` to your `config/custom_components/` directory.

## Setup

1. Go to **Settings → Integrations → Add Integration**
2. Search for "Solarseed TOU"
3. Select your energy sensor (supports kWh, Wh, W, kW)
4. Go to the [Rate Calculator](https://johnnysolarseed.org/tou-calculator) and generate your YAML
5. Go to **Settings → Integrations → Solarseed TOU → Configure**
6. Paste the generated YAML and click Submit

## The Rate Formula

The effective $/kWh rate for any tier is:

```
effective_rate = (
    tier_rate                   # summed usage + transmission + distribution + PCA
  + regulatory_per_kwh          # summed flat per-kWh regulatory surcharges
  + state_passthrough_per_kwh   # summed state-mandated per-kWh items
  + programs_per_kwh            # summed enrolled programs
) × (1 + tax_rate_pct / 100)   # summed percentage-based taxes
```

The `tier_rate` varies by time of day. Everything else is shared across all tiers. The calculator computes these values from your bill — you don't need to do the math.

## Sensors Created

| Sensor | Description |
|--------|-------------|
| `sensor.solarseed_tou_current_rate` | Active effective $/kWh rate (full formula) |
| `sensor.solarseed_tou_current_tier` | Name of active tier (for automations) |
| `sensor.solarseed_tou_fixed_monthly` | Fixed monthly charge from config ($/mo) |
| `sensor.solarseed_tou_cost_per_hour` | Instantaneous cost rate ($/hr) |
| `sensor.solarseed_tou_cost_today` | Accumulated cost since midnight |
| `sensor.solarseed_tou_cost_this_week` | Since Monday |
| `sensor.solarseed_tou_cost_this_month` | Since 1st of month |

### Current Rate Attributes

The `current_rate` sensor exposes the full formula breakdown:

| Attribute | Description |
|-----------|-------------|
| `tier_id` | Active tier ID (e.g., "on-peak") |
| `tier_name` | Human-readable tier name |
| `tier_base_rate` | Bare tier rate before adders/tax |
| `regulatory_per_kwh` | Regulatory surcharges |
| `state_passthrough_per_kwh` | State pass-through charges |
| `programs_per_kwh` | Enrolled programs contribution |
| `tax_rate_pct` | Combined tax percentage |
| `fixed_monthly` | Monthly fixed charge |
| `is_holiday` | Whether today is a holiday |
| `next_rate_change` | ISO datetime of next tier change |
| `next_tier` | Name of next tier |

## Automations

Trigger automations on tier changes:

```yaml
trigger:
  - platform: state
    entity_id: sensor.solarseed_tou_current_tier
    to: "On-Peak"
action:
  - service: notify.notify
    data:
      message: "On-peak rates are now active!"
```

## Upgrading from v0.6.x

v0.7.0 removes the built-in GUI panel and Lovelace card. Rate configuration is now done exclusively via YAML from the website calculator.

Existing configurations will be automatically migrated — your current rates will continue to work. The new formula fields (`regulatory_per_kwh`, `state_passthrough_per_kwh`, `programs_per_kwh`, `tax_rate_pct`) default to 0, so your existing `rate` values are used as-is until you re-export from the calculator.

To take advantage of the full formula breakdown, re-run your bill through the [Rate Calculator](https://johnnysolarseed.org/tou-calculator) and paste the new YAML.
